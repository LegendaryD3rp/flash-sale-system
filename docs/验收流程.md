# 验收展示流程

## 第一部分：基础支撑功能（5分钟）

### 1.1 用户注册与登录（前端展示）
- 打开前端 `http://localhost:3000/`
- 注册两个账号：普通用户 `testuser` / `admin`
- 登录后JWT Token自动存储，页面跳转首页
- 展示不同角色的权限差异（普通用户看不到管理后台）

### 1.2 商品浏览与管理（Swagger + 前端）
- 用Postman/Swagger添加5个商品（管理员）
- 前端首页展示商品列表、分页、搜索
- 点进商品详情页，查看价格、描述、评价

### 1.3 下单与支付（前端）
- 普通用户加购 → 结算 → 下单 → 支付
- 查看订单列表和订单状态变化

---

## 第二部分：核心架构功能（10分钟）

### 2.1 统一网关入口（抓包展示）
- 用浏览器开发者工具/Postman观察请求头
- 所有请求 `→ Gateway:8080 → 路由到各服务`
- 展示Gateway限流效果：超限返回429

### 2.2 高并发防超卖 — 万人秒杀演示（命令行）
```bash
cd e2e
python3 wanren_tongchang.py --mode seckill --orders 10000
# 或更精简版:
python3 wanren_tongchang.py --mode seckill --orders 10000 --verify basic --cleanup light
```
- 开跑前先展示Redis库存（`seckill:stock:{id}`）
- 跑完后展示Redis库存为0（零超卖）
- 展示MySQL订单数=10000（库存一致）

### 2.3 异步订单削峰（MQ监控）
- 秒杀请求瞬间返回"排队中"
- 展示RabbitMQ队列堆积与消费过程
- 对比同步下单vs异步削峰的响应时间差异

### 2.4 WebSocket实时推送（前端确认）
- 前端打开秒杀页面，WebSocket自动连接
- 秒杀成功后，前端实时收到"订单创建成功"通知
- 对比前端轮询方案：WebSocket推送0延迟 vs 轮询延迟

### 2.5 AI智能导购（前端对话）
- 打开AI导购页面 `/ai-chat`
- 提问："推荐一款适合打游戏的手机"
- AI根据商品描述+大模型生成推荐回复

---

## 第三部分：技术亮点总结（2分钟）

| 技术点 | 实现方案 | 验证结果 |
|--------|---------|---------|
| 网关控制 | Spring Cloud Gateway + JWT + 限流 | 全部通过Gateway路由 |
| 防超卖 | Redis DECR原子扣减 + SADD防重复 | 10000并发0超卖 |
| 削峰 | RabbitMQ异步落单 | 秒杀瞬时响应<50ms |
| 实时推送 | WebSocket ConcurrentHashMap | 订单成功即时通知 |
| AI导购 | DeepSeek API + MQ排队 | 流式回复流畅 |
